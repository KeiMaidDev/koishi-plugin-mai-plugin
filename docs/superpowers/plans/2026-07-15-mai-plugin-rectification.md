# Mai Plugin Rectification Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the plugin emit valid actionable QQ raw Markdown, expose explicit LXNS OAuth bind/unbind commands, build cleanly through the root Koishi Yakumo pipeline, and publish only required runtime files.

**Architecture:** Keep QQ protocol construction in `src/platform/qq-message.ts`, command composition in command modules, and OAuth state/token ownership in `UpdateService` plus repositories. Complete Tasks 1-4 sequentially with a red-green test cycle and one independent commit per Task; Task 5 is verification and reporting only.

**Tech Stack:** TypeScript, Koishi 4, Satori elements, Vitest 3, Yarn 4, Yakumo tsc/esbuild, Node.js 18+.

## Global Constraints

- Do not rewrite query, rendering, guessing, queueing, or score algorithms.
- Use test-first implementation: observe each targeted test fail before changing production code.
- Do not dispatch subagents because the required 5.6 model version cannot be guaranteed.
- Actionable QQ guidance must contain non-empty `qq:rawmarkdown` content and a keyboard.
- Non-QQ platforms and compatibility mode must retain readable text/image fallbacks.
- Never log or echo OAuth code, state, access token, refresh token, client secret, import token, or cipher key.
- URL actions accept only absolute `https:` URLs; command actions use plugin-owned fixed commands.
- Do not delete runtime fonts, generated/fallback assets, test fixtures, render baselines, or asset-generation scripts.
- Never commit `node_modules`, `lib`, `dist`, or `tsconfig.tsbuildinfo`.
- Run the production build from `C:\koishi-app` with `yarn build mai-plugin`.

## File Structure

- `src/platform/qq-message.ts`: QQ button/keyboard protocol types, constructors, validation, and rich/fallback sending.
- `src/platform/fallback-message.ts`: query error classification and human-readable fallback copy.
- `src/commands/support.ts`: command reply helpers and reusable guidance builders.
- `src/commands/help.ts`, `settings.ts`, `music.ts`, `image.ts`, `guess.ts`: scenario-specific fixed buttons and pagination composition.
- `src/commands/update.ts`, `core.ts`: explicit LXNS bind/unbind commands and compatibility aliases.
- `src/services/update-service.ts`, `src/server/callback-store.ts`: single-use OAuth state lifecycle and unbind cleanup.
- `src/database/repositories.ts`, `src/providers/lxns.ts`: encrypted token persistence/removal.
- `src/render/assets.ts`, `package.json`, `tsconfig.json`: CommonJS runtime asset resolution and Yakumo package contract.
- `readme.md`, `.gitignore`: user-facing operation/build docs and local-artifact policy.
