# 配置项中文说明设计

## 目标

为 Koishi 控制台中的所有公开插件配置补充清晰的中文分组标题和字段说明，使管理员无需查阅源码即可完成配置。

## 兼容边界

- 保留现有英文配置键、数据结构、类型、默认值和校验范围。
- 不迁移或重写现有 YAML 配置。
- 不改变插件运行时行为。
- Koishi 表单仍以英文配置键作为字段标题，中文说明显示在字段下方，形成中英对照。

## Schema 设计

直接在 `src/config.ts` 的现有 Schemastery 定义上添加中文 `.description()`：

- 为 `developerTokens`、`oauth`、`resourceSync` 和 `render` 添加中文分组说明。
- 为所有叶子字段添加用途说明。
- 对时间字段明确单位，对并发和队列字段说明限制含义。
- 对令牌、密钥、公开地址和主机白名单说明安全或格式要求。
- `publicBaseUrl`、`administrators` 和 `compatibilityMode` 也必须有中文说明。

不引入额外本地化文件或多语言字典。插件当前面向中文 maimai 使用场景，直接中文说明更简单，也符合仓库现有 Schema 写法。

## 验证

- 扩展现有 Schema 集成测试，递归检查每个公开字段都包含非空中文说明。
- 保留现有默认值、范围和 secret role 断言。
- 运行配置相关测试、完整测试和生产构建。
