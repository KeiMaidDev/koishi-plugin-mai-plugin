# README 重写设计

## 目标

参照 `mai-bridge` README 的信息组织方式，重写 `koishi-plugin-mai-plugin` 的 README，让安装者能够快速了解插件用途、完成安装与基础配置，并找到常用命令和开发构建方式。

## 文档结构

README 按以下顺序组织：

1. 项目简介与主要能力
2. 安装方式与插件包名
3. 运行要求
4. 基础配置及关键配置说明
5. 查分器绑定与成绩更新
6. 常用命令
7. 回调地址与兼容模式
8. 项目结构
9. 开发构建
10. 许可证

## 内容边界

- 所有配置、依赖服务、命令和路径以当前源码与 `package.json` 为准。
- 保留 OAuth、公开回调地址、数据安全和代理部署所必需的提醒，但不展开成长篇运维手册。
- 项目结构按目录和模块职责说明，覆盖 `src`、`assets` 及根目录关键文件，不逐一解释每个源文件。
- 不写入插件包中不存在的 `scripts`、测试工具或独立 TypeScript 开发依赖。
- 开发构建只记录 Koishi 根工作区的 `yarn build` 流程。

## 验收

- README 的章节路径与参考项目一致地简洁、面向安装和使用。
- 包名、服务依赖、默认配置、命令入口和回调路径与当前实现一致。
- 包含清晰的项目结构树及目录职责说明。
- Markdown 标题、表格和代码块结构有效。
- 根工作区 `yarn build` 通过，且不覆盖当前工作区中与本任务无关的修改。
