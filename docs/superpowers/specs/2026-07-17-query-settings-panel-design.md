# 查分设置面板设计

## 目标

新增独立指令 `/mai 查分设置`，使用 QQ Raw Markdown 下挂 keyboard，为用户集中展示和调整头像、牌子、查分器以及落雪/水鱼绑定状态。原有 `/mai 设置mai` 与 `/mai 设置b50` 继续可用，并显示同一面板。

## 面板内容

Raw Markdown 正文显示以下当前值：

- 查分器：自动、水鱼或落雪。
- 头像：当前头像 ID；未设置时显示“默认”。
- 牌子：当前牌子 ID；未设置时显示“默认”。
- 落雪：已绑定或未绑定。
- 水鱼：已绑定或未绑定。

Keyboard 使用三行：

1. `设置头像`、`设置牌子`。按钮填入 `/mai 设置头像 ` 或 `/mai 设置牌子 `，不自动发送，等待用户补充 ID 或名称。
2. `自动`、`水鱼`、`落雪`。按钮分别执行现有查分器切换指令并立即发送。
3. 落雪绑定按钮、水鱼绑定按钮。两个按钮始终存在，各自根据实时绑定状态显示绑定或解绑操作。

所有命令按钮不限制点击用户，且 `reply` 固定为 `false`。

## 绑定按钮状态

| 落雪 | 水鱼 | 按钮 |
| --- | --- | --- |
| 未绑定 | 未绑定 | `绑定落雪`、`绑定水鱼` |
| 已绑定 | 未绑定 | `解绑落雪`、`绑定水鱼` |
| 未绑定 | 已绑定 | `绑定落雪`、`解绑水鱼` |
| 已绑定 | 已绑定 | `解绑落雪`、`解绑水鱼` |

绑定落雪继续执行 `/mai 绑定落雪` 并进入现有 OAuth 授权流程。解绑落雪继续执行 `/mai 解绑落雪`。绑定水鱼执行 `/mai 绑定水鱼 `，只填入命令并等待用户补充 Token；解绑水鱼执行新增的 `/mai 解绑水鱼` 并立即发送。

## 服务边界

- 设置服务负责读取当前查分器、头像和牌子。
- 更新服务提供落雪和水鱼的绑定状态，并负责解绑水鱼。
- 设置指令只负责组织面板内容和按钮，不直接访问数据库。
- 水鱼解绑只清除当前用户保存的导入 Token，不影响 QQ 绑定、成绩数据或查分器偏好。

## 错误处理

- 面板状态读取失败时返回现有“设置失败，请稍后重试”提示，不发送状态不完整的 keyboard。
- 解绑水鱼失败时复用更新指令的失败提示。
- 未绑定状态下执行解绑应保持幂等，仍返回解绑成功。

## 验收

- `/mai 查分设置`、`/mai 设置mai` 和 `/mai 设置b50` 均输出 Raw Markdown keyboard 面板。
- 四种落雪/水鱼绑定组合均生成正确的两个按钮及对应命令。
- 头像和牌子按钮不自动发送；查分器切换和解绑按钮立即发送。
- 所有命令按钮的 `reply` 为 `false`，且不设置指定用户或管理员权限。
- `/mai 解绑水鱼` 清除 Token，刷新面板后按钮变为“绑定水鱼”。
- Koishi 根目录 `yarn build` 通过。
