# 远端歌曲别名缓存设计

## 目标

当本地远端别名缓存不存在或没有有效歌曲别名时，从稳定的远端数据源获取完整歌曲别名表并持久缓存。歌曲搜索同时使用歌曲标题、远端基础别名和用户维护别名。

用户通过指令添加、投票和删除的别名继续存储在 `mai_alias` 数据表，不与远端缓存混写。

## 数据源

当前仅使用落雪公开接口：

```text
GET https://maimai.lxns.net/api/v0/maimai/alias/list
```

响应结构：

```json
{
  "aliases": [
    {
      "song_id": 8,
      "aliases": ["别名一", "别名二"]
    }
  ]
}
```

水鱼公开 `music_data` 不包含别名，且目前未发现稳定的公开别名端点，因此本次不构造或猜测水鱼 URL。以后水鱼提供稳定端点时，可扩展为“落雪、水鱼、本地缓存”的远端回退链。

## 存储边界

- 远端基础别名保存在 `resourceSync.cacheDir/aliases.json`。
- 文件使用独立 schema，不加入游戏资源 snapshot manifest，避免别名接口故障导致歌曲、头像和牌子数据源整体回退。
- 写入使用同目录临时文件和原子重命名，进程中断时不留下半写缓存。
- 用户别名继续保存在 `mai_alias` 和 `mai_alias_vote`。
- 远端同步不得创建、修改或删除用户别名和投票记录。

## 缓存格式

```json
{
  "schemaVersion": 1,
  "source": "lxns",
  "generatedAt": "2026-07-18T00:00:00.000Z",
  "aliases": [
    {
      "musicId": 8,
      "names": ["别名一", "别名二"]
    }
  ]
}
```

解析和归一化规则：

- `schemaVersion` 必须为 `1`。
- `musicId` 必须是正安全整数，并且存在于当前 `MaimaiDataStore.musics`。
- `names` 必须是字符串数组。
- 每个别名去除首尾空白；丢弃空字符串、控制字符和长度超过 128 个 Unicode code point 的值。
- 同一歌曲内按标准化搜索文本去重，保留第一个有效显示文本。
- 同一歌曲的多个远端条目合并。
- 解析后没有任何有效别名视为无可用缓存。

## 启动流程

1. 完成当前歌曲资源启动，得到 `MaimaiDataStore`。
2. 尝试读取并解析 `aliases.json`。
3. 缓存包含有效别名时直接使用，不访问远端。
4. 缓存不存在、格式错误或解析后为空时，请求落雪完整别名表。
5. 结构化校验落雪响应并按当前歌曲表归一化。
6. 至少包含一个有效别名时原子写入缓存，并发布到 `MaimaiDataStore`。
7. 远端请求、解析或写入失败时记录警告并使用空远端别名集合；插件继续启动。

损坏缓存不得覆盖为新的空文件；有效旧缓存不得因远端错误被删除。

## 搜索合并

`AliasService` 为每首歌曲构造搜索字段时按以下顺序合并：

1. 歌曲正式标题。
2. `MaimaiDataStore` 中的远端基础别名。
3. `mai_alias` 中票数非负的用户别名。

完全相同或标准化后相同的字段只保留一次。现有标题精确匹配、别名精确匹配、模糊匹配和包含匹配顺序不变。

远端别名只参与搜索，不参与添加、删除、投票或票数展示。管理员删除同名用户别名不会删除远端基础别名。

## 安全与限制

- 只允许请求固定的落雪 HTTPS 地址，不接受用户配置 URL 或重定向到其他主机。
- 请求沿用 `resourceSync.timeoutMs`，响应体上限为 8 MiB。
- 最多接受 20,000 个歌曲条目、每首 128 个别名，防止异常响应耗尽内存。
- 所有远端数据均按 `unknown` 解析并逐字段校验，不直接类型断言。
- 调试日志只记录来源、收到/保留条目数、缓存命中和错误类型，不记录具体别名内容。

## 验收

- 有效本地缓存存在时不访问落雪，搜索可命中缓存别名。
- 缓存缺失、损坏或为空时请求落雪并原子缓存有效结果。
- 落雪返回未知歌曲、重复别名、空白、控制字符、超长文本和超量数据时正确过滤或拒绝。
- 落雪请求、响应解析和缓存写入失败均不阻塞插件启动。
- 用户别名与远端别名同时参与精确和模糊搜索，且用户投票数据不被修改。
- 管理员删除用户别名不会删除同名远端别名。
- 调试日志不包含具体别名。
- `package.json` 不新增 `scripts`、`engines` 或测试工具。
- Koishi 根目录 `yarn build` 通过。
